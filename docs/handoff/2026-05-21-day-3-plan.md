### 2. Resolve Fast Naming Decisions

Recommended decisions for today:

- Use `Operations Hub` for the logged-in home label. The existing implementation plan and copy tests already use it.
- Keep `Report in` as an authenticated action, not the public signup CTA. Use `Create account` and `Log in` publicly.
- Lead public landing copy with `Star Citizen operations`; use `across the 'verse` in supporting copy.
- Treat fan-kit source folders as local references for now. Move only approved production assets into `src/assets/...`.
- Do not wait for the full visual brand guide before starting the shared foundation. Build the shell conservatively with the current industrial direction, then refine visuals after the foundation route model exists.

### 3. Start The Staffing Review Track

Import this workbook into Google Sheets:

```text
outputs/position-review/star-citizen-ship-position-review-one-line-preserved.xlsx
```

Review priority:

1. Include/exclude each ship.
2. Confirm Skeleton crew counts.
3. Confirm Standard crew counts.
4. Confirm Full Crew counts.
5. Flag ships requiring special doctrine notes.

Reason: staffing templates are a known data-quality blocker for Fleet Command, but this work can run alongside shared-foundation implementation.

### 4. Start The Icon Asset Track

Use `docs/mockups/musterdeck-free-icon-candidates-v3.html` as the source of truth for the locked icon set.

First production-prep tasks:

- Vendor the approved SVGs locally under a new mockup asset subfolder first, not `src/assets`.
- Add an attribution register for Apache 2.0, MIT, ISC, and CC BY icons.
- Keep user-supplied references marked separately from free-library icons.
- Build one final locked-icon proof sheet before moving anything into production.

Reason: the icon choices are now mostly stable, but licensing and source tracking need to be clean before production use.

### 5. Begin Shared Foundation Implementation

Create a dedicated branch when coding starts:

```bash
git switch -c codex/shared-foundation
```

Execute `docs/superpowers/plans/2026-05-20-shared-foundation-implementation.md` in order.

Recommended Day 3 implementation checkpoint:

1. Task 1: foundation route model.
2. Task 2: brand voice copy constants.
3. Task 4: foundation demo data.
4. Stop and verify tests/build before adding UI components.

Defer Task 5 UI components until the route, copy, and demo data modules are passing tests. Defer deeper auth/OAuth behavior to a follow-up plan, as the current shared-foundation plan already recommends.
